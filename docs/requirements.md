# 📚 나만의 독서 기록 & 지식 베이스 — 요구사항 명세서

> v0.1 · 2026년 3월

---

## 1. 프로젝트 개요

### 1.1 배경 및 목적

노션으로 독서 기록을 정리하다가 추가 기능 구현의 한계를 느낌.
단순 기록을 넘어 개발 작업 시 바로 참조할 수 있는 **개인 지식 베이스**를 직접 개발하고자 함.

**주요 목적**

- 독서 내용을 구조적으로 기록하고 관리
- 프로그래밍 등 기술 서적의 내용을 개발 시 바로 참조
- 나만의 RAG(Retrieval-Augmented Generation) 기반 지식 베이스 구축
- 노션 의존 없이 완전히 커스터마이징 가능한 나만의 도구

### 1.2 프로젝트 성격

- 현재: 1인 사용자
- 향후: 멀티유저로 확장 가능한 구조로 설계

## 2. 기술 스택

## 3. 핵심 개념 및 데이터 모델

### 3.1 주요 엔티티

| 엔티티 | 설명 |
|--------|------|
| `Book` | 제목, 저자, ISBN, 장르, 독서 상태 등 |
| `Note` | 책에 연결되거나 독립적으로 존재 가능한 자유 기록 |
| `Highlight` | 책의 인용구 + 페이지 번호 별도 관리 |
| `Tag` | 노트/책에 자유롭게 붙이는 분류 레이블 |

### 3.2 Book - Note 관계

- 책 한 권에 노트 여러 개 자유롭게 첨부 가능
- 챕터/섹션 구분은 노트 제목 또는 태그로 관리
- 책 없이 독립적인 자유 노트도 생성 가능 (개념 정리, 메모 등)

## 4. 기능 요구사항

### 4.1 책 관리

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| ISBN/제목 자동 검색 | 외부 API로 책 정보 자동 입력 | 필수 |
| 수동 입력 | ISBN 없는 책이나 직접 입력 | 필수 |
| 독서 상태 관리 | 읽는 중 / 완독 / 읽고 싶음 | 필수 |
| 책 목록 뷰 | 표지, 제목, 저자, 상태 한눈에 보기 | 필수 |

### 4.2 노트 작성

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 마크다운 에디터 | 자유로운 마크다운 작성, 코드블록 지원 | 필수 |
| 정해진 템플릿 | 책 제목, 핵심 요약, 메모 등 구조화된 양식 | 필수 |
| 기술 노트 | 코드 스니펫 포함 가능한 기술 서적 전용 노트 | 필수 |
| 인용구 + 내 생각 | 인용구와 개인 코멘트 분리 작성 | 필수 |
| 독립 자유 노트 | 책 없이도 자유롭게 노트 생성 | 필수 |

### 4.3 하이라이트

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 인용구 저장 | 책의 특정 구절, 페이지 번호와 함께 저장 | 필수 |
| 내 코멘트 추가 | 인용구에 개인 생각/해석 덧붙이기 | 필수 |
| 하이라이트 모아보기 | 책별 또는 전체 하이라이트 리스트 뷰 | 중요 |

### 4.4 태그 & 검색

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 자유 태그 | 노트/책에 태그 자유롭게 붙이기 | 필수 |
| 키워드 검색 | 제목, 내용, 태그 기반 텍스트 검색 | 필수 |
| 태그별 필터링 | 태그로 노트/책 필터링 | 중요 |
| 의미 기반 검색 (RAG) | AI 임베딩 기반 유사 내용 검색 | 선택 |

### 4.5 AI 독서 대화 (소크라테스식 정리)

> 책 내용을 AI와 대화하며 생각을 구체화하고, 그 결과를 노트로 저장하는 기능

**흐름**

```
책 내용 입력/붙여넣기
      ↓
AI가 소크라테스식 질문으로 대화 유도
(한 주제를 깊이 파고드는 스타일)
      ↓
대화 종료 후 노트 저장
  ├── 📄 원본 대화 전문
  └── 📝 AI 요약 노트 (마크다운 형태)
```

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 내용 입력 | 책 내용을 직접 입력하거나 붙여넣기 | 중요 |
| 소크라테스식 대화 | AI가 한 주제를 깊이 파고드는 질문 연속 제시 | 중요 |
| 대화 → 노트 저장 | 원본 대화 전문 + AI 요약본 함께 저장 | 중요 |
| 요약 노트 편집 | 저장 전 AI 요약 내용 사용자가 수정 가능 | 선택 |

### 4.6 사용자 & 인증

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 1인 사용자 (1단계) | 로그인 없이 단일 사용자로 시작 | 필수 |
| 회원가입/로그인 (2단계) | JWT 기반 인증, 멀티유저 대비 구조 설계 | 중요 |
| 멀티유저 (3단계) | 사용자별 독립된 데이터 관리 | 선택 |

## 5. 비기능 요구사항

- **반응형 UI**: 모바일, 태블릿, 데스크탑 모두 사용 가능
- **마크다운 렌더링**: 작성/미리보기 전환 또는 실시간 렌더링
- **코드 하이라이팅**: 기술 노트의 코드블록 언어별 하이라이팅
- **확장성**: 1인 → 멀티유저 전환을 고려한 DB 및 API 설계
- **pgvector 준비**: 처음부터 pgvector로 구성해 AI 기능 전환 용이하게

## 6. 개발 로드맵

| 단계 | 목표 | 주요 기능 |
|------|------|-----------|
| **1단계 (MVP)** | 핵심 기능 완성 | DB 설계, Spring Boot CRUD API, React 기본 UI, 책 등록, 노트 작성, 마크다운, 태그, 키워드 검색 |
| **2단계 (고도화)** | 편의 기능 추가 | ISBN 자동 검색, 하이라이트 관리, 템플릿, 코드 하이라이팅, 모바일 반응형 완성, **AI 소크라테스식 독서 대화 + 노트 저장** |
| **3단계 (AI/RAG)** | 지식 베이스화 | OpenAI 임베딩, pgvector 벡터 검색, AI 질의응답 (내 노트 기반) |
| **4단계 (멀티유저)** | 서비스 확장 | JWT 인증, 회원가입/로그인, 사용자별 데이터 분리 |
